是的，**常量（Constant）** 的核心定义就是**其值在程序运行期间不会改变**。

这是一个非常重要的概念，它有两个主要目的：

1.  **保证数据安全**：防止你或其他开发者在代码中意外地修改一个不应该被修改的值。
2.  **增强代码可读性**：用一个有意义的名字（如 `PI`）代替一个“魔法数字”（如 `3.14159`），让代码更容易理解。

---

### C++ 中定义常量的两种主要方式

#### 1. `const` 关键字

`const` 是最常用、最灵活的定义常量的方式。它可以用来修饰变量、函数参数、函数返回值等。

**特点：**
*   **有类型**：`const` 常量是有数据类型的（如 `const int`, `const double`），编译器可以进行类型检查。
*   **有作用域**：`const` 常量的作用域遵循普通变量的规则。如果在函数内部定义，它只在该函数内有效；如果在全局作用域定义，它在整个文件中有效（或通过 `extern` 在其他文件中共享）。
*   **可以在运行时初始化**：`const` 变量的值可以在运行时确定，但一旦确定后就不能再改变。

**示例：**
```cpp
#include <iostream>

int main() {
    const double PI = 3.14159; // 定义一个名为 PI 的常量
    int radius = 10;

    double area = PI * radius * radius;
    std::cout << "圆的面积是: " << area << std::endl;

    // PI = 3.14; // 编译错误！不能修改 const 常量的值
    return 0;
}
```

#### 2. `#define` 预处理指令

`#define` 是一个来自 C 语言的宏定义指令，它在**预处理阶段**工作。它的作用是进行简单的文本替换。

**特点：**
*   **无类型**：`#define` 定义的宏没有数据类型，它只是一个文本替换。这可能导致一些意想不到的类型错误。
*   **无作用域**：`#define` 定义的宏从定义点到文件末尾都有效（除非被 `#undef` 取消定义）。它不遵循块作用域规则，容易造成命名冲突。
*   **必须在编译时确定值**：宏的值必须是一个编译时常量。

**示例：**
```cpp
#include <iostream>

#define PI 3.14159 // 定义一个名为 PI 的宏

int main() {
    int radius = 10;

    // 在编译前，代码中的 PI 会被直接替换成 3.14159
    double area = PI * radius * radius;
    std::cout << "圆的面积是: " << area << std::endl;

    return 0;
}
```

---

### `const` vs `#define`：现代 C++ 的选择

在现代 C++ 编程中，**强烈推荐使用 `const` 而不是 `#define` 来定义常量**。

| 特性 | `const` 变量 | `#define` 宏 |
| :--- | :--- | :--- |
| **类型检查** | **有**，更安全 | **无**，只是文本替换，容易出错 |
| **作用域** | **有**，遵循块作用域，可控 | **无**，从定义处到文件结尾都有效，易冲突 |
| **调试** | **可以**，调试器能显示 `const` 变量的值 | **不可以**，宏在预处理阶段已被替换，调试器看不到 |
| **代码维护** | **容易**，语义清晰 | **困难**，宏替换可能导致意想不到的副作用 |

**简单来说，`const` 是一个真正的、有类型、有作用域的常量，而 `#define` 只是一个盲目的文本替换工具。** 除非有特殊的宏编程需求（如条件编译 `#ifdef`），否则都应该优先使用 `const`。

### 特殊情况：`const_cast`

有一个非常高级且危险的操作叫做 `const_cast`，它可以“去掉”一个变量的 `const` 属性。

**示例：**
```cpp
#include <iostream>

int main() {
    const int a = 10;
    // int* p = &a; // 编译错误！不能把 const int* 赋值给 int*

    // 使用 const_cast "去掉" const 属性
    int* p = const_cast<int*>(&a);

    std::cout << "修改前 a 的值: " << a << std::endl; // 输出 10

    *p = 20; // 危险！尝试通过指针修改 const 变量的值

    std::cout << "修改后 a 的值: " << a << std::endl; // 输出什么？
    std::cout << "指针 p 指向的值: " << *p << std::endl; // 输出什么？
    
    return 0;
}
```
这段代码的行为是**未定义的（Undefined Behavior）**。在某些编译器和优化级别下，第一个 `cout` 和第二个 `cout` 可能都会输出 `10`，因为编译器可能已经将常量 `a` 的值直接替换到代码中了。这证明了试图修改常量是多么危险和不可靠。

**结论：**
请始终将 `const` 视为一个**不可违背的承诺**。常量就是用来保证其值不变的，任何试图绕过这个规则的行为都应该被避免。
